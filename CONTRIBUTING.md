# Contributing to Gemini CLI Git Tools Extension

I would love to accept your contributions to this project. By contributing, you agree to abide by our [Code of Conduct](./CODE_OF_CONDUCT.md).

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue on our [GitHub Issues page](https://github.com/paufortiana/geminicli-git-tools-extension/issues). Please include as much detail as possible, including:

*   A clear and descriptive title.
*   A step-by-step description of how to reproduce the bug.
*   The expected behavior and what actually happened.
*   Your environment details (e.g., OS, Gemini CLI version).

### Suggesting Enhancements

If you have an idea for a new feature or an improvement to an existing one, please open an issue on our [GitHub Issues page](https://github.com/paufortiana/geminicli-git-tools-extension/issues). This allows us to discuss the idea before you start working on it.

### Pull Requests

I welcome pull requests! Here's how to submit one:

1.  **Fork the repository** and create your branch from `main`.
2.  **Set up your development environment**. This is a Gemini CLI extension, so you'll need to have the Gemini CLI installed.
3.  **Make your changes**. Please ensure your code adheres to the existing style and conventions.
4.  **Write a clear and concise commit message** that follows the [Conventional Commits specification](https://www.conventionalcommits.org/).
5.  **Open a pull request** to the `main` branch of the repository.

### Code Reviews

All submissions, including submissions by project members, require review. I use [GitHub pull requests](https://docs.github.com/articles/about-pull-requests) for this purpose.

## Development Setup

This project is a Gemini CLI extension. To get started with development:

1.  **Install the Gemini CLI**: Follow the [official installation instructions](https://github.com/google-gemini/gemini-cli?tab=readme-ov-file#-installation).
2.  **Clone your fork**: `git clone https://github.com/YOUR_USERNAME/geminicli-git-tools-extension.git`
3.  **Navigate to the project directory**: `cd geminicli-git-tools-extension`

Now you can start making changes to the extension's files.

## Coding Standards

*   **Conventional Commits**: All commit messages must adhere to the [Conventional Commits specification](https://www.conventionalcommits.org/).
*   **TOML Files**: Command definitions and prompts are configured using TOML files. Please maintain the existing structure and style.
